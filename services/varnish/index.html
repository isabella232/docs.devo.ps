Varnish Official documentation

Commands

start
Start Varnish if stopped
stop
Stop Varnish if started
reload
Reload Varnish, reload the configuration and perform a graceful restart
restart
Restart Varnish, reload the configuration (but kills existing connection)

Configuration

string admin_listen_address Optional, Default: 127.0.0.1
Telnet admin interface listen address, usually only bind localhost IP address for the admin interface.
integer admin_listen_port Optional, Default: 6082
Telnet admin interface listen port.
string listen_address Optional
Blank address means all IPv4 and IPv6 interfaces, otherwise specify a host name, an IPv4 dotted quad.
integer listen_port Optional, Default: 6081
Listen port.
integer max_threads Optional, Default: 1000
The maximum number of worker threads to start.
integer min_threads Optional, Default: 1
The minimum number of worker threads to start.
string secret_file Optional, Default: /etc/varnish/secret
File containing administration secret.
string storage_file Optional, Default: /var/lib/varnish/$INSTANCE/varnish_storage.bin
Storage file, only used if storage_type is file.
string storage_size Optional, Default: 256M
Cache size: in bytes, optionally using k / M / G / T suffix.
string storage_type Optional, Default: malloc
Store objects either in memory (malloc), on in a file (file).
integer thread_timeout Optional, Default: 120
Idle timeout for worker threads.
integer ttl Optional, Default: 120
Default TTL used when the backend does not specify one.
string vcl_conf Optional, Default: /etc/varnish/default.vcl
Main configuration file.
