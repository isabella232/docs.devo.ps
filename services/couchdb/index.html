CouchDB Official documentation

Commands

start
Start CouchDB if stopped
stop
Stop CouchDB if started
reload
Reload CouchDB, reload the configuration and perform a graceful restart
restart
Restart CouchDB, reload the configuration (but kills existing connection)

Notes

For better user experience you may prefer the use of the reload task instead of restart.

Configuration

admin_pass string admin_pass
Define the admin password Default: admin
admin_user string admin_user
Define the admin username Default: admin
bind_address string bind_address
The IP address to bind to. Default: 127.0.0.1
enable integer enable
Define whether CouchDB should start at bood time Default: 1
insecure boolean insecure
Allow insecure setup (no admin user)
max_document_size integer max_document_size
Maximum document size Default: 4294967296
port integer port
Listening port Default: 5984
running_user string running_user
Define the running user for the CouchDB process (it needs to exist) Default: couchdb
stderr_file string stderr_file
Where to send the process STDERR Default: /dev/null
stdout_file string stdout_file
Where to send the process STDOUT Default: /dev/null
