MySQL

The world's most popular open source relational database

Example

services:
  mysql: '*'
configuration:
  mysql:
    bind_address: 0.0.0.0
    innodb:
      innodb_buffer_pool_size: 8G

Install MySQL on the node, make it listen on all interfaces (0.0.0.0) and allocate 8GB of RAM to the InnoDB buffer pool.

Configuration

  • port

    Connection port.

    Type Default Required
    integer 3306 false

  • bind_address

    Binding IP address.

    Type Default Required
    string 127.0.0.1 false

  • socket

    Mysql listening socket.

    Type Default Required
    string /var/run/mysqld/mysqld.sock false

  • pid_file

    Pid file.

    Type Default Required
    string /var/run/mysqld/mysqld.pid false

  • base_dir

    Base folder.

    Type Default Required
    string /usr false

  • data_dir

    Database files path.

    Type Default Required
    string /var/lib/mysql false

  • auto_increment_increment

    Auto increment.

    Type Default Required
    integer 1 false

  • auto_increment_offset

    Auto increment offset.

    Type Default Required
    integer 1 false

  • old_passwords

    Allow old (non-secure) passwords.

    Type Default Required
    integer 0 false

  • tunable

    • key_buffer (tunable.key_buffer)

      Size of the buffer used for index blocks. The key buffer is also known as the key cache.

      Type Default Required
      string 256M false

    • max_allowed_packet (tunable.max_allowed_packet)

      The maximum size of one packet or any generated/intermediate string.

      Type Default Required
      string 16M false

    • thread_stack (tunable.thread_stack)

      The stack size for each thread.

      Type Default Required
      string 256K false

    • thread_cache_size (tunable.thread_cache_size)

      Number of threads the server should cache for reuse.

      Type Default Required
      integer 32 false

    • max_connections (tunable.max_connections)

      The maximum permitted number of simultaneous client connections.

      Type Default Required
      integer 256 false

    • wait_timeout (tunable.wait_timeout)

      The number of seconds the server waits for activity on a noninteractive connection before closing it.

      Type Default Required
      integer 180 false

    • net_read_timeout (tunable.net_read_timeout)

      The number of seconds to wait for more data from a connection before aborting the read.

      Type Default Required
      integer 30 false

    • net_write_timeout (tunable.net_write_timeout)

      The number of seconds to wait for a block to be written to a connection before aborting the write.

      Type Default Required
      integer 30 false

    • back_log (tunable.back_log)

      The number of outstanding connection requests MySQL can have. In other words this value is the size of the listen queue for incoming TCP/IP connections.

      Type Default Required
      integer 128 false

    • table_open_cache (tunable.table_open_cache)

      The number of open tables for all threads.

      Type Default Required
      integer 1024 false

    • max_heap_table_size (tunable.max_heap_table_size)

      Maximum size to which user-created MEMORY tables are permitted to grow.

      Type Default Required
      string 64M false

  • query_cache

    • query_cache_limit (query_cache.query_cache_limit)

      Do not cache results that are larger than this number of bytes.

      Type Default Required
      string 2M false

    • query_cache_size (query_cache.query_cache_size)

      The amount of memory allocated for caching query results.

      Type Default Required
      string 64M false

  • logs

    • log_error (logs.log_error)

      The location of the error log.

      Type Default Required
      string /var/log/mysql/mysql.log false

    • slow_query_log (logs.slow_query_log)

      Whether the slow query log is enabled.

      Type Default Required
      integer 1 false

    • slow_query_log_file (logs.slow_query_log_file)

      The name of the slow query log file.

      Type Default Required
      string /var/log/mysql/slow.log false

    • long_query_time (logs.long_query_time)

      Increment slow_queries variable and log if a query takes longer than this many seconds.

      Type Default Required
      number 1 false

    • log_queries_not_using_indexes (logs.log_queries_not_using_indexes)

      If you are using this option with the slow query log enabled then queries that are expected to retrieve all rows are logged.

      Type Default Required
      string false false

  • replication

    • server_id (replication.server_id)

      Is used in replication to enable master and slave servers to identify themselves uniquely.

      Type Default Required
      integer 10 false

    • log_bin (replication.log_bin)

      Enable binary logging. The option value is the basename for the log sequence.

      Type Default Required
      string mysql-bin false

    • log_slave_updates (replication.log_slave_updates)

      This option tells the slave to log the updates performed by its SQL thread to its own binary log.

      Type Default Required
      string true false

    • log_bin_trust_function_creators (replication.log_bin_trust_function_creators)

      Controls whether stored function creators can be trusted not to create stored functions that will cause unsafe events to be written to the binary log.

      Type Default Required
      string false false

    • expire_logs_days (replication.expire_logs_days)

      The number of days for automatic binary log file removal.

      Type Default Required
      integer 10 false

    • max_binlog_size (replication.max_binlog_size)

      If a write to the binary log causes the current log file size to exceed the value of this variable then the server rotates the binary logs.

      Type Default Required
      string 100M false

    • relay_log (replication.relay_log)

      The basename for the relay log.

      Type Default Required
      string relay-bin false

    • sync_binlog (replication.sync_binlog)

      If the value of this variable is greater than 0 then the MySQL server synchronizes its binary log to disk (using fdatasync()) after every sync_binlog writes to the binary log.

      Type Default Required
      integer 1 false

    • skip_slave_start (replication.skip_slave_start)

      Tells the slave server not to start the slave threads when the server starts.

      Type Default Required
      string true false

  • innodb

    • innodb_buffer_pool_size (innodb.innodb_buffer_pool_size)

      The size in bytes of the memory buffer InnoDB uses to cache data and indexes of its tables.

      Type Default Required
      string 128M false

    • innodb_log_file_size (innodb.innodb_log_file_size)

      The size in bytes of each log file in a log group.

      Type Default Required
      string 5M false

    • innodb_additional_mem_pool_size (innodb.innodb_additional_mem_pool_size)

      The size in bytes of a memory pool InnoDB uses to store data dictionary information and other internal data structures.

      Type Default Required
      string 8M false

    • innodb_data_file_path (innodb.innodb_data_file_path)

      The paths to individual data files and their sizes.

      Type Default Required
      string ibdata1:10M:autoextend:max:500M false

    • innodb_flush_log_at_trx_commit (innodb.innodb_flush_log_at_trx_commit)

      If 0 the log buffer is written out to the log file once per second. If 1 the log buffer is written out to the log file at each transaction commit and the flush to disk operation is performed on the log file. If 2 the log buffer is written out to the file at each commit but the flush to disk operation is not performed on it.

      Type Default Required
      integer 1 false

    • innodb_flush_method (innodb.innodb_flush_method)

      Flush method for InnoDB, either of fdatasync, O_DSYNC or O_DIRECT

      Type Default Required
      string O_DIRECT false

    • innodb_log_buffer_size (innodb.innodb_log_buffer_size)

      The size in bytes of the buffer that InnoDB uses to write to the log files on disk.

      Type Default Required
      string 8M false

    • innodb_adaptive_flushing (innodb.innodb_adaptive_flushing)

      Specifies whether to dynamically adjust the rate of flushing dirty pages in the InnoDB buffer pool based on the workload.

      Type Default Required
      string true false
